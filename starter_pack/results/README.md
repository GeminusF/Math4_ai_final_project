## 1. Synthetic Datasets

### 1.1 Linear Gaussian

| Model | Test Acc | Test CE |
|-------|----------|---------|
| Softmax Regression | 0.9500 | 0.1543 |
| Neural Network (h=32) | 0.9500 | 0.1532 |

Both models reach the same accuracy, which makes sense since the Bayes-optimal
boundary for two Gaussians with shared covariance is a straight line. The NN
adds parameters but doesn't actually help here.

### 1.2 Moons

| Model | Test Acc | Test CE |
|-------|----------|---------|
| Softmax Regression | 0.8500 | 0.2854 |
| Neural Network (h=32) | 0.9750 | 0.1013 |

The NN clearly wins here -- it can wrap its boundary around the crescent shapes
thanks to the tanh hidden layer. The softmax model is stuck with a straight line
through the interleaving arcs.

## 2. Digits Benchmark

| Model | Optimizer | Test Acc | Test CE | Best Epoch |
|-------|-----------|----------|---------|------------|
| Softmax | SGD 0.05 | 0.9402 | 0.2694 | 200 |
| NN (h=32) | Adam 0.001 | 0.9511 | 0.1507 | 198 |

About 1 ppt accuracy gain and cross-entropy drops by almost half.
The NN also converges noticeably faster based on the training curves.

**Figures:** `digits_{softmax,nn}_curves.png`

## 3. Capacity Ablation (Moons)

Widths {2, 8, 32} with Adam lr=0.005, 300 epochs:

| Width | Test Acc | Test CE |
|-------|----------|---------|
| 2 | 0.8625 | 0.2502 |
| 8 | 0.9750 | 0.0853 |
| 32 | 0.9750 | 0.0904 |

Width 2 is basically as limited as the linear model -- only two oriented
"soft" partitions through tanh. At width 8 the boundary becomes genuinely
curved and test accuracy jumps to 97.5%. Width 32 doesn't improve further
on this simple task (it's already saturated).

**Figure:** `moons_capacity_ablation.png`

## 4. Optimizer Study (Digits NN, h=32)

| Optimizer | Test Acc | Test CE | Best Epoch |
|-----------|----------|---------|------------|
| SGD (lr=0.05) | 0.9484 | 0.1700 | 198 |
| Momentum (lr=0.05, mu=0.9) | 0.9538 | 0.1653 | 198 |
| Adam (lr=0.001) | 0.9511 | 0.1507 | 198 |

Momentum gets the best accuracy (95.4%), Adam gets the lowest CE (0.151).
Looking at the curves, Momentum converges way faster than plain SGD in
the first ~20 epochs. Adam is a bit slower early on but reaches better
final calibration.

**Figure:** `digits_optimizer_comparison.png`

## 5. Repeated Seeds (Digits, 5 seeds, 95% CI)

### Softmax (SGD lr=0.05)

| Seed | Test Acc | Test CE |
|------|----------|---------|
| 0 | 0.9402 | 0.2694 |
| 1 | 0.9402 | 0.2695 |
| 2 | 0.9402 | 0.2694 |
| 3 | 0.9375 | 0.2693 |
| 4 | 0.9402 | 0.2694 |

**Result:** Acc = **0.9397 ± 0.0015**, CE = **0.2694 ± 0.0001**

### Neural Network (Adam lr=0.001, h=32)

| Seed | Test Acc | Test CE |
|------|----------|---------|
| 0 | 0.9511 | 0.1630 |
| 1 | 0.9620 | 0.1388 |
| 2 | 0.9511 | 0.1739 |
| 3 | 0.9674 | 0.1548 |
| 4 | 0.9538 | 0.1527 |

**Result:** Acc = **0.9571 ± 0.0091**, CE = **0.1566 ± 0.0162**

The CIs for cross-entropy don't overlap at all ([0.269, 0.270] vs [0.140, 0.173]),
so we can be fairly confident the NN genuinely beats the linear model here. The
softmax results are almost identical across seeds (it starts at zero, so only the
batch order changes), while the NN varies more because of random weight init.

| Bin | Count | Avg Conf | Avg Acc |

| [0.20, 0.40] | 1 | 0.360 | 0.000 |

The NN puts 92.7% of predictions in the highest confidence bin and its accuracy

under-confident in the middle bins -- 83% accuracy in the [0.4, 0.6] range
where it only claims ~52% confidence.

Both models show that wrong predictions have much lower confidence and higher
entropy, which means these metrics are actually useful for flagging uncertain
predictions.
